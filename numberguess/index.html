If you collect the user's input as a string (i.e. without using parseInt() to turn it into an integer), does == work to compare the input to the stored secret number? Would === work? Which is best?

Using == would work, because it will "coerce" the value into a number to compare it. Since we're doing an operation on the number in the string, JS will change it to an integer for that comparison. If you use a === operator, that checks for strict equality, and that would return false, since a string of a number and the integer of that number are not strictly equal.

If you collect the user's input as an integer (i.e. by using parseInt() to turn it into an integer), can you use == to compare? ===? Which is better in this case?

If you parse the value into an integer, you can use both == and === to compare them. In this case, I would guess === is better because you are comparing two integers, and since it's the same data type, strict comparison is probably better

What is the difference between == and ===?

== is a loose comparison, allowing for things close in value to be evaluated as equal, like 6 and "6". === is a strict comparison, where the values must be identical for it to evaluate as true.
