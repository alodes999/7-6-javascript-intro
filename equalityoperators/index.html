console.log( "5 == '5' is " + (5 == '5') ); //true The operator is an equals to, but not a strict one, and since strings with numbers can be operated on, integer 5 and string 5 are loosely equal, so it returns true

console.log( "5 == 'five' is " + (5 == 'five') ); //false The same operator returns false here because the word five would not be a string of the number 5, therefore it wouldn't be able to be operated on.

console.log( "5 == '6' is " + (5 == '6') ); //false This operator does try and compare equality between integer and number in a string, but 5 is not equal to 6, so it's false

console.log( "5 === '5' is " + (5 === '5') ); //false This operator is also equals to, but it is a strict equality. Therefore, an integer of 5 is not the same as a number in a string 5, so it's false

console.log( "5 == 2 + 3 is " + (5 == 2 + 3) ); //true This operator does return true because 2 + 3 is equal to 5.

console.log( "5 == 5 is " + (5 == 5) ); //true These are the same, so true

console.log( "5 === 5 is " + (5 === 5) ); //true These again, are strictly the same, so even with the 3 ='s it will return true.

What do you call true and false values?

They are boolean values.

How is true similar or dissimilar from "true"? Can they be used in the same way?

true is a boolean value, "true" is a string. They cannot be used in the same way, because when you run true == "true", it returns false.
